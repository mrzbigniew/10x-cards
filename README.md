## Project Description

Responsive web application for creating, managing, and studying flashcards. The project combines manual flashcard authoring with AI-generated flashcard candidates, giving users full control over what gets saved before starting a simple study session in a single app.

- Create and activate a user account.
- Sign in securely with email and password.
- Create flashcards manually or generate AI-based flashcard candidates from source text.
- Review, edit, accept, or reject AI-generated candidates before saving them.
- Manage private flashcards with list views, pagination, and CRUD operations.
- Start a simple study session directly from the flashcard list.

The product solves the problem of time-consuming manual flashcard creation by helping users move quickly from source material to ready-to-study cards without losing editorial control over the content generated by AI.

## Tech Stack

### Frontend

- `Astro 6` for a fast, performant web application architecture.
- `React 19` for interactive UI components.
- `TypeScript 5` for static typing and better maintainability.
- `Tailwind CSS 4` for utility-first styling.
- `shadcn/ui` and `Radix UI` primitives for accessible UI components.

### Backend

- `Supabase` as the backend platform.
- `PostgreSQL` database provided through Supabase.
- Built-in Supabase authentication for account management and access control.

### AI

- `OpenRouter` for communication with multiple LLM providers and models.

### Tooling and Delivery

- `ESLint` and `Prettier` for code quality and formatting.
- `Husky` and `lint-staged` for pre-commit automation.
- `GitHub Actions` planned for CI/CD pipelines.
- `DigitalOcean` planned for Docker-based hosting.

## Getting Started Locally

### Prerequisites

- `Node.js 24.14.0` as defined in `.nvmrc`
- `npm` as the package manager (`package-lock.json` is present)

### Installation

```bash
npm install
```

### Start the development server

```bash
npm run dev
```

Then open the local URL printed by Astro in your terminal.

### Build for production

```bash
npm run build
```

### Preview the production build

```bash
npm run preview
```

### Notes

- The repository currently includes the application scaffold and core frontend tooling.
- Backend services, AI integration, and deployment configuration are part of the product plan, but their runtime setup details are not fully documented in the provided files yet.

## Project Scope

### In Scope for the MVP

- User registration, email-based account activation, sign-in, password reset, and password change.
- Private flashcard storage with one logical flashcard set per user.
- Flashcard list view with empty state, pagination, metadata, and contextual actions.
- Manual flashcard creation, editing, and deletion.
- AI-assisted flashcard candidate generation from user-provided text.
- Candidate review flow with edit, reject, and atomic accept actions.
- Basic study session in a modal using an open-source repetition library.
- AI generation logging for quality analysis.
- Responsive UI and accessibility aligned with `WCAG 2.2 AA`.

### Out of Scope for the MVP

- A custom advanced spaced-repetition algorithm comparable to Anki or SuperMemo.
- Importing multiple file formats such as PDF or DOCX.
- Sharing flashcard sets between users.
- Integrations with external learning platforms.
- Native mobile applications.
- Extended onboarding flows.
- Advanced product analytics beyond AI generation logs.
- Detailed legal, privacy, and support processes not defined in the PRD.

## Project Status

The project is in the `MVP planning and early implementation` stage.

At this point:

- The product requirements, scope, and success metrics are defined in the PRD.
- The technical direction and frontend foundation are documented and scaffolded.
- The repository currently reflects an Astro-based starter setup rather than a fully implemented product.
- CI/CD and hosting targets have been identified, but final delivery pipelines are still to be completed.

This means the repository is suitable for active development, but it should not yet be considered feature-complete or production-ready.
